# Práctica en Clase 05 - Despliegue en Azure

## Descripción

Esta práctica explica cómo configurar los recursos en Azure necesarios para publicar una Web API de .NET 8, utilizando la suscripción estudiantil gratuita y GitHub Actions para la automatización del despliegue.

---

## 1 - Crear una base de datos en Azure SQL (suscripción estudiantil)

### Requisito previo

Activar la suscripción gratuita para estudiantes en:  
🔗 https://azure.microsoft.com/pt-br/free/students  
Inicia sesión con tu correo institucional (`.edu`). Obtienes **USD 100 en crédito** sin tarjeta de crédito.

---

### Pasos

**1. Ingresar al Portal de Azure**

Ir a https://portal.azure.com e iniciar sesión con la cuenta estudiantil.

---

**2. Crear un servidor SQL**

1. Buscar **"SQL servers"** en la barra de búsqueda.
2. Clic en **+ Create**.
3. Completar los campos:

| Campo | Valor |
|-------|-------|
| Subscription | Azure for Students |
| Resource group | Crear nuevo (ej. `rg-practica`) |
| Server name | Nombre único global (ej. `servidor-curso`) |
| Location | East US *(región más cercana disponible)* |
| Authentication | SQL authentication |
| Admin login | `sqladmin` |
| Password | Contraseña segura (anótala) |

4. Clic en **Review + create** → **Create**.

---

**3. Crear la base de datos**

1. Ir al servidor recién creado → **+ Create database**.
2. Completar:

| Campo | Valor |
|-------|-------|
| Database name | Nombre de tu BD (ej. `MiApiDB`) |
| Compute + storage | **Free** (32 MB, sin costo) — si no aparece, usar **Basic** (5 DTU / 2 GB) |
| Backup storage redundancy | Locally-redundant |

3. Clic en **Review + create** → **Create**.

---

**4. Configurar el firewall**

1. En el servidor SQL → **Networking**.
2. Activar **"Allow Azure services and resources to access this server"** ← permite que App Service se conecte.
3. En la sección **Firewall rules**, clic en **"Add your client IPv4 address"** ← agrega automáticamente tu IP actual para que puedas conectarte desde tu equipo (SSMS, Azure Data Studio, etc.).
4. Guardar.

---

**5. Obtener la cadena de conexión**

1. Ir a la base de datos → **Connection strings** → pestaña **ADO.NET**.
2. Copiar la cadena — se usará como secreto en GitHub (`AZURE_SQL_CONNECTION_STRING`).

---

## 2 - Crear un App Service para .NET 8

### Pasos

**1. Crear el App Service**

1. Buscar **"App Services"** en el portal.
2. Clic en **+ Create** → **Web App**.
3. Completar los campos:

| Campo | Valor |
|-------|-------|
| Subscription | Azure for Students |
| Resource group | El mismo del paso anterior |
| Name | Nombre único global (ej. `mi-api-curso`) — este será el subdominio |
| Publish | **Code** |
| Runtime stack | **.NET 8 (LTS)** |
| OS | **Linux** *(más económico en plan gratuito)* |
| Region | East US |

4. En **Pricing plan** seleccionar **Free F1** (disponible en suscripción estudiantil).
5. Clic en **Review + create** → **Create**.

---

**2. Obtener el Publish Profile**

1. Ir al App Service creado → **Settings** → **Configuration** → pestaña **General settings**.
2. En **SCM Basic Auth Publishing Credentials** activar **On** y guardar.  
   > ⚠️ Sin este paso el botón de descarga estará deshabilitado.
3. Ir a **Overview**.
4. Clic en **Download publish profile**.
3. Abrir el archivo descargado con un editor de texto y copiar **todo el contenido**.
4. En GitHub: repositorio → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**.
   - Nombre: `AZURE_WEBAPP_PUBLISH_PROFILE`
   - Valor: pegar el contenido del publish profile.

---

**3. Configurar la cadena de conexión (opcional)**

Si la API necesita conectarse a la BD del paso PC05.1:

1. En el App Service → **Configuration** → **Connection strings**.
2. Agregar nueva entrada:
   - Name: `DefaultConnection`
   - Value: cadena de conexión de Azure SQL
   - Type: `SQLAzure`
3. Guardar y reiniciar el servicio.

## 3 - Crear el workflow de GitHub Actions para despliegue de la BD

Crear el archivo `.github/workflows/deploy-db.yml` en el repositorio usando como base el workflow de referencia:

📄 [Actions/Ejemplos/BDDeploy.yml](../Actions/Ejemplos/BDDeploy.yml)

### Requisito previo — secreto en GitHub

En el repositorio → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**:

| Nombre | Valor |
|--------|-------|
| `AZURE_SQL_CONNECTION_STRING` | Cadena de conexión obtenida en PC05.1 paso 5 |

### Ajustes necesarios antes de usar el YML

| Variable | Qué cambiar |
|----------|------------|
| `DACPAC_PATH` | Ruta al `.dacpac` generado por tu proyecto de BD |
| `paths: 'BD/**'` | Carpeta donde vive tu proyecto `.sqlproj` |

> 💡 El workflow solo se ejecuta cuando hay cambios dentro de la carpeta `BD/`. Así el despliegue de la API y el de la BD son independientes.

## 4 - Crear el workflow de GitHub Actions para despliegue de la API

Crear el archivo `.github/workflows/deploy-api.yml` en el repositorio usando como base el workflow de referencia:

📄 [Actions/Ejemplos/APIDeploy.yml](../Actions/Ejemplos/APIDeploy.yml)



---

### Ajustes necesarios antes de usar el YML

| Variable | Qué cambiar |
|----------|------------|
| `AZURE_WEBAPP_NAME` | Nombre exacto de tu App Service |
| `AZURE_WEBAPP_PACKAGE_PATH` | Ruta a la carpeta del proyecto `.csproj` del API |
| `AZURE_WEBAPP_PUBLISH_PROFILE` | Secreto en GitHub con el publish profile descargado |

> 💡 Si el proyecto tiene múltiples capas (como en PC03), el `dotnet restore` en la raíz de la solución restora todos los proyectos referenciados automáticamente.
