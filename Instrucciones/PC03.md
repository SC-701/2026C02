# Práctica en Clase 03 - Implementación CRUD de Álbum

## 1. Descripción

A partir del código base desarrollado en la **Práctica en Clase 02** — el cual contiene la estructura de la API con todos los métodos sin implementar debe llevar a cabo la implementación funcional del CRUD para la entidad **Álbum**.

No se requiere aplicar validaciones en esta práctica. El objetivo es que los métodos del controlador respondan correctamente según los conocimientos adquiridos hasta la **Semana 03**.

---

## 2. Objetivo

Implementar los métodos del CRUD de la entidad Álbum en una Web API de .NET, de forma que cada endpoint sea funcional y retorne una respuesta HTTP adecuada.

## 5. Requerimientos

### 5.1. Métodos a implementar

Se deben implementar los siguientes endpoints en el controlador de Álbum.

### 5.2. Consideraciones técnicas

- Implementar los métodos en **todas las capas** involucradas: controlador, flujo  y acceso a datos.
- Utilizar los **atributos correctos** para recibir los parámetros (`[FromRoute]`, `[FromBody]`, `[FromQuery]`, según corresponda).
- Retornar los **códigos de estado HTTP** apropiados para cada operación:
  - `200 OK` — consulta exitosa
  - `201 Created` — recurso creado
  - `204 No Content` — actualización o eliminación exitosa
  - `404 Not Found` — recurso no encontrado
- Las respuestas deben ser devueltas mediante **Action Methods** (`IActionResult` o `ActionResult<T>`).
- El contenido de las respuestas debe estar en **formato JSON**.
- Definir **rutas claras** en cada endpoint mediante el atributo `[Route]` o los atributos de método (`[HttpGet]`, `[HttpPost]`, etc.).
- No se requiere persistencia en base de datos; se puede utilizar una **lista en memoria** para almacenar los álbumes durante la ejecución.

### 5.3. Lo que NO se requiere en esta práctica

- Validaciones de modelo (`[Required]`, `[MaxLength]`, etc.)
- Autenticación o autorización
